Сьогодні ми познайомимось з системою керування версій git, програму мало лиш написати її також потрібно доставити користувачу.

Система керування версіями — інструмент, який дозволяє одночасно, не заважаючи один одному, проводити роботу над груповими проектами.

Спершу вам потрібно встановити програму git на комп'ютери.
https://git-scm.com/

Далі вам потрібно скачати цей репозиторій, для цього створіть нову папку і відкрийте там командний рядок, введіть там наступну команду
git clone https://github.com/FusionMDD/Git_lesson.git

Команда git clone дозволяє скачати копію репозиторію

Після скачування ознайомтесь з матеріалами і виконайте команду git pull 

git pull дозволяє оновити вміст вашого локального репозиторію до актуальної версії.

git підтримує роботу з різними гілками
Галуження - це відмежування від основної лінії розробки для продовження своєї частини роботи та уникнення конфліктів з основною лінією.

https://git-scm.com/book/uk/v2/%D0%93%D0%B0%D0%BB%D1%83%D0%B6%D0%B5%D0%BD%D0%BD%D1%8F-%D0%B2-git-%D0%93%D1%96%D0%BB%D0%BA%D0%B8-%D1%83-%D0%BA%D1%96%D0%BB%D1%8C%D0%BA%D0%BE%D1%85-%D1%81%D0%BB%D0%BE%D0%B2%D0%B0%D1%85

Щоб переглянути наявні гілки введіть команду git branch

Щоб створити нову гілку введіть git checkout -b, ви створите нову гілку і перейдете на неї
git checkout це команда що дозволяє переміщатись між різними гілками та комітами, прапорець -b дозволяє створити нову гілку
Вам потрібно відгілкуватись від гілки main і створити власну гілку назвавши її child_branch

Перегляньте дерево комітів виконавши команду git log 
Коміт у git — це об'єкт, що містить посилання на:
	знімок стану (snapshot), що записаний раніше в індекс
	метадані автора
	коментарі
	нуль чи більше вказівників на інші коміти, що є прямими батьками даного коміту

https://uk.wikipedia.org/wiki/%D0%9A%D0%BE%D0%BC%D1%96%D1%82_(%D0%BA%D0%B5%D1%80%D1%83%D0%B2%D0%B0%D0%BD%D0%BD%D1%8F_%D0%B2%D0%B5%D1%80%D1%81%D1%96%D1%8F%D0%BC%D0%B8)

Додайте текстовий файл до вашого локального репозиторію, в файлі напишіть ваш розклад уроків на сьогодні.

Виконайте команду git status 

git status дозволяє побачити змінені файли в вашому репозиторії та відстежувати файли заготовані для наступного коміту

Виконайте команду git add . 
Команда git add додає файли які будуть додані в наступний коміт, аргументом команди є область яку потрібно додати в майбутній коміт, . означає що ми хочемо додати усі файли з даної папки.

Створіть коміт командою git commit, в полі повідомлення опишіть файли які ви додаєте
Команда git commit створює об'єкт з файлів обраних у попередньому кроці

Змініть ваш текстовий файл додавши в нього ваш розклад уроків на завтра. І закомітьте зміни використовуючи набуті знання.
